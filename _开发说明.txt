
因为 vscode 插件在打包时会直接把 README.md 的内容显示在插件的发布页面上，所以其中并不方便发表一些额外的开发信息，只能作为用户使用小手册。

所以开发相关的说明放到这里来吧。

--------------------------------------------------------

程序本身比较简单，集中在 src/extension.ts 就完了。

不过 vscode 插件相关的东西比较啰嗦，不过好在也没有大坑。基本上开发可以参照网上其他大神的文章，下面主要说下发布时的一些不大不小的坑。

程序功能主要是给每个项目一个监听 http 服务器，接受请求后可调用 vscode 打开指定文件的某行。

这是为了将来集成 ui 设计 ide 准备的，也是为了代码快速浏览开发器准备的，很显然大家开发有自己的小设计器和 ide 也可以这样集成上超级强大的 vscode 的代码编辑器。

这样主要是可以分离设计器和 vscode ，全部用插件式的来开发未免效率太低 -- 对于不熟悉的开发来说。再说了谁也指不定 vscode 什么时候也被其他的新事物代替啊。所以没必要绑那么紧。

发布时其实是可以直接在 

https://marketplace.visualstudio.com/vscode

中发布的，我询问 ai 老是要我申请 Azure DevOps 账户，这大概是在老外的环境下比较方便吧。对于国内那是坑。

登录后可以直接关联 github 的账号，比较方便，不过小坑是这里生成一个新插件的时候国内网络环境直连的话是没反应的。不知道是不是这样，反正我发布是卡这里。

过了这里后面的其实就简单了，另外它也有一个小审核。这里还有个小坑，老是说我的 auth 和 id 对不上，我一直以为 auth 是我 github 的账号或者是在扩展商店中的账号。其实都不是，
